目前价值体系由积分代替，因为没有任何收益。
目前名称还是叫价值系统。

积分的产出分为：
固定产出：活动产生（细化产生的规则）。
评估产出：对项目每个阶段主要人员做进一步贡献评估。

价值系统设计：
1. 积分可追踪，积分值的由来，在组织或项目中产生的活动日志都要有记录。用于核对积分
2. 积分类型：目前想到两种：1. 组织积分。 2. 项目积分
3. 评估系统

组织积分，由组织内部活动产生的积分。用于组织分利。
1. 组织内推积分奖励。
2. 人员培训积分
3. 组织扩大推广积分
4. 文档知识库固定积分
5. idea提出积分，idea采纳积分（评估）
6. 组织内部管理积分
7. 组织文化，规则优化整理积分
8. ...

项目积分，在具体项目中产生的积分。用于项目分利
1. 项目积分要具体项目具体分析。
2. 项目负责人积分
3. 项目推动进行积分
4. 项目阶段性评估积分。(定期评估)
5. ...

评估系统，主要用于项目和组织阶段性评估。
1. 项目评估，在项目进行阶段中，每个阶段对项目成员积分进行评估。（主要为前中期还没收益的时候）
评估方式：1. 项目成员与组织核心成员（项目管理核心）一起评估组织内部成员贡献，给出具体的积分分配（先用估值，在进行百分比分配具体积分数量）。
2. 组织评估
组织评估由组织核心成员和负责组织各个模块的成员进行评估。这里不设计百分比，而是用具体的积分数字。


价值和利益流向：
1. 组织的全部利益来自组织内部的成员和外部关联的项目。
2. 所以组织需要完全把控项目，而不能因为项目起飞，由项目内的成员去控制项目脱离组织。（因某些原因确实需要脱离，需要组织内所有核心成员评估）
3. 项目的主要负责人，都至少是组织核心成员。核心成员必须控制项目利益的百分比流入组织内部。
4. 核心成员退出组织需要移交项目负责人身份（如果有），但是项目里的贡献和组织的贡献积分不会清理。
5. 负责人的项目积分一半由组织积分代替。同时组织需要占项目利益固定百分比（看具体规划）。根据具体的项目积分和组织积分吃到两份分利。

风险控制与治理
防止刷分、低质刷贡献

明确“可衡量指标”，对“固定产出”类活动要求提供切实成果（文档链接、培训录像、推广效果数据等）；避免无实质内容的“刷活跃”。

评估产出需由多人（项目核心成员+组织评估小组）审核，不允许单一负责人或推荐人自行分配高分。

定期审计积分记录，发现异常行为（如短期内大量低价值提交、多人互相推荐但无实际效果）时启动调查，并可能扣减积分或取消资格。

争议解决流程

建立“积分争议申诉”机制：成员若对自己积分计算或评估结果有异议，可提交申诉，由专门评估小组复核并给出书面反馈。

申诉流程与时限需明确（如申诉需在结果公布后一周内提出，复核在两周内完成）。

规则调整与治理结构

设计“积分规则委员会”或“价值体系维护小组”，由若干核心成员和代表成员组成，定期（如每季度）审查规则执行情况，提出调整建议并组织表决或共识形成。

重大规则修订需事前公示、收集反馈，待多数核心成员或持特定资格成员同意后生效。

保密与数据安全

积分系统涉及成员贡献与组织策略，不宜全部公开给外部；内部需要权限分层，敏感数据（如个人绩效细节）仅限相关人员可见。

确保所用工具或平台的数据备份与安全性，避免日志或积分记录丢失。

持续改进与复盘

定期回顾价值体系运行效果：成员满意度调查、激励效果分析、组织增长状况等；根据结果优化积分规则、评估流程和工具。

组织内部举行分享会或调研，收集成员对价值系统的意见与建议，及时迭代。
